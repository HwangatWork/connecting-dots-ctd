## 1. 제품 철학 & 핵심 원칙

### 1.1 시스템 정의

> **"점들을 연결해서 판단을 만든다"**

커넥팅닷은 한국 개인 투자자를 위한 실시간 투자 판단 시스템입니다. 64개 지표를 수집하고 계산하지만, 유저에게는 판단 결과만 보여줍니다. 숫자가 아니라 의미를 전달합니다.

### 1.2 핵심 설계 원칙

| 원칙 | 의미 | UX 구현 방식 |
|---|---|---|
| **결론이 먼저** | 판단 문장이 숫자보다 크게 보인다 | Hero 영역: 온도 숫자보다 "분할 매수 구간" 텍스트가 먼저 읽힘 |
| **숫자보다 의미** | 62점보다 "분할 매수 구간"이 행동을 만든다 | 아크 게이지, 레이더 형태, 수평 CTD 체인으로 시각화 |
| **점들의 연결** | 인과관계가 보여야 판단을 신뢰한다 | CTD 수평 체인 5노드 → 클릭 시 상세 드릴다운 |
| **세후 기준** | 국내/해외 비교는 세후가 실제 수익이다 | Zone 4 매수플랜 + 종목 비교에 세후 수익률 자동 반영 |
| **8개 이하 동시 노출** | Miller's Law — 동시 노출 지표 최대 8개 | 레이더 8축, 퀵 메트릭 6칩, 드릴다운으로 단계 분리 |

### 1.3 유저의 핵심 질문 3개

```
시장 탭:    "지금 시장이 어떤 상태인가?"  → 30초 안에 답
종목 탭:    "이 종목 지금 사도 되나?"     → 클릭 1회로 전체 판단
나의 투자:  "내 투자 세후로 얼마 남나?"   → 자동 계산으로 즉시 확인
```

## 2. 브랜드 & 디자인 언어

### 2.1 포지셔닝

> **"Bloomberg의 신뢰감 + Apple의 완성도 + Perplexity의 판단 흐름"**

- **Bloomberg Terminal:** 정보 밀도, 전문가 신뢰감, 모노스페이스 숫자 언어
- **Apple macOS Dark:** 4단계 레이어 배경, blur 공간감, Inter 타이포 정밀도
- **Perplexity AI:** 결론 우선 흐름, AI 판단의 신뢰 표현, 인과관계 서사

### 2.2 디자인 무드

```
어둡고 정밀하다. 신뢰가 간다. 군더더기가 없다.
숫자가 아름답다. 판단이 보인다.
```

### 2.3 확정된 디자인 결정

- **다크 전용** — Light mode 미지원. 투자 집중력과 데이터 시각화 가시성.
- **강조색 단 하나** — Apple Blue `#0a84ff`. 선택 상태·링크·스코어에만.
- **폰트 역할 분담** — Inter(문장) + JetBrains Mono(모든 숫자·가격·지표값) + Noto Sans KR(한국어 폴백)
- **그라데이션·그림자 금지** — 레이어 색상 차이로만 깊이 표현

### 2.4 금지 원칙

```
❌ 밝은 배경
❌ 강조색 2개 이상 동시 사용
❌ drop-shadow / glow / neon
❌ 동시 노출 지표 9개 이상
❌ 배지 언어 Zone별 독립 정의
❌ 세로 CTD 체인 (수평 방식 B로 확정)
```

## 5. 64개 지표 — 2트랙 노출 전략

### 5.1 핵심 설계 철학

> **"64개를 보여주는 게 아니라 64개를 활용하는 것"**

64개 지표는 화면에 나열하는 데이터가 아니라 **판단을 만드는 엔진의 부품**입니다. 유저가 보는 것은 엔진의 출력값만입니다.

### 5.2 트랙 분류

| 트랙 | 카테고리 | 지표 수 | 노출 방식 |
|---|---|---|---|
| **Track A** | 시장 지수 | 6 | 메트릭 바 칩 2개 표면 노출 |
| **Track A** | 매크로 | 6 | CTD 01단계 체인 텍스트 |
| **Track A** | 심리 | 6 | 온도 게이지 입력값, VIX·F&G 칩 |
| **Track A** | 수급 | 3 | Zone 1B 수급 바 전체 노출 |
| **Track B** | 펀더멘털 | 9 | 레이더 성장·품질·재무 축 → 드릴다운 |
| **Track B** | 밸류에이션 | 8 | 레이더 밸류에이션 축 → 드릴다운 |
| **Track B** | 기술적 | 8 | 레이더 타이밍 축 → 드릴다운 |
| **Track B** | 세금·절세 | 10 | Zone 4 세후 인라인 + 나의 투자 탭 |
| **프레임** | 종목 평가 8항목 | 8 | 레이더 8축 자체 |

### 5.3 레이더 8축 ↔ 지표 매핑

| 레이더 축 | 색상 | 연결 지표 | 드릴다운 수 |
|---|---|---|---|
| 비즈니스 품질 | Accent Blue | 영업이익률·Gross Margin·FCF·시장점유율 | 4개 |
| 성장 모멘텀 | Green | EPS·매출 YoY·AI CapEx 수혜·FCF | 7개 |
| 밸류에이션 | Purple | P/E·PEG·EV/EBITDA·DCF·P/Book 등 | 8개 ✅ 완전 |
| 시장 타이밍 | Yellow | RSI·Stoch RSI·MA50·MA200·볼린저·Beta | 6개 ✅ 완전 |
| 재무 건전성 | Accent Blue | 부채비율·유동비율·현금성자산·FCF | 4개 |
| 매크로 연계 | Green | 외국인·기관·개인 수급 + AI CapEx | 5개 |
| 리스크 관리 | Yellow | VIX·SKEW·Put/Call·Beta | 4개 |
| 세후 수익률 | Purple | 양도세·배당세·ISA 효과·실효세율 | 4개 |

**커버리지:** 64개 중 54개 레이더 포함. 미포함 10개는 세금·절세 전용 (나의 투자 탭).

### 5.4 CTD 체인 커버리지 (확인된 빈틈)

```
현재 CTD에 포함된 지표:   약 32개
빈틈 ① — 매크로 6개 중 4개 미반영:
  장단기금리차, 하이일드스프레드, WTI, 연준 총자산
  → CTD 01단계에 추가 예정

빈틈 ② — 시장 지수 6개가 CTD 결론과 미연결:
  코스피/나스닥 Drawdown → 국면 판단 → CTD 03 결론
  → 명시적 연결 추가 예정
```

## 6. 웹 레이아웃 구조

### 6.1 고정 레이아웃 치수

```
Topbar:      height 52px, position fixed, z-index 200

메트릭 바:   height 54px, position sticky, top 52px, z-index 90
메인 영역:   margin-left 260px
최대 너비:   없음 (전체 너비 활용)
```

### 6.2 Topbar

```
배경: rgba(0,0,0,.85) + backdrop-filter blur(20px) saturate(180%)
하단 테두리: .5px solid rgba(255,255,255,.08)

구성 (좌→우):
  로고: "CTD" 16px 700 -.03em, "IQ" = Accent Blue
  구분선: .5px solid rgba(255,255,255,.15), height 18px
  탭: [● 시장] [종목] [나의 투자] — 13px 500
  우측: 갱신 시각 11px mono + 새로고침 버튼 28px 원형

탭 상태:
  기본:  color --t3, background transparent
  호버:  color --t2, background --g1
  활성:  color --t1, background --g2
  활성+점: 탭명 앞 5px 원형 도트 (시장 = --gr)
```

### 6.3 사이드바 (탭별 콘텐츠)

**시장 탭 사이드바 — CTD 체인:**
```
섹션 레이블: "Zone 2 — CTD 체인"
스텝 3개 (01 매크로 환경 / 02 수급 신호 / 03 최종 결론)

스텝 기본 상태: 클릭 시 해당 스텝만 펼쳐짐 (나머지 닫힘)
펼쳐진 상태:
  체인 행: → 화살표 + 텍스트 + 배지
  결론 박스: --acd 배경, 12px 텍스트
```

**종목 탭 사이드바 — Top 5 리스트:**
```
상단: 강세장 국면 배지 (크게, --grd 배경)
```

| Perplexity AI | 결론 우선, 퍼플 | 판단 흐름 표현 방식 | 영감 참고 |

### 12.2 노션형 시스템에서 차용 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 데이터 신선도 표시 | **차용** — 메트릭 바에 통합 | 투자 시스템 신뢰도 필수. 별도 탭 대신 칩으로 |
| 종목 탭 국면 카드 격상 | **차용** — 메인 상단으로 | 국면→종목 인과 연결 강화 |
| 수평 CTD 체인 | **차용 + 개선** — 방식 B 확정 | 전체 논리 1초 파악. 클릭 상세 추가 |
| 흰 배경 | **차용 안 함** | 다크 UI 확정 |
| 19개 목차 앵커 | **차용 안 함** | 스크롤 최소화 원칙 위배 |
| 데이터 갱신 전용 탭 | **차용 안 함** | Miller's Law 위반 |
| 체크박스 논거 | **차용 안 함** | 배지 시스템이 더 직관적 |

### 12.3 CTD 표현 방식 비교 결정

```
방식 A (수직 아코디언):
  장점: 상세 텍스트 충실, 깊이 있게 읽기 좋음
  단점: 전체 흐름 파악에 끝까지 읽어야 함

방식 B (수평 흐름 + 클릭 상세):
  장점: 전체 논리 구조 1초에 파악, 인터랙티브
  단점: 노드당 텍스트 제한

→ 방식 B 확정. 이유: "처음 보는 사람이 3초 안에 구조가 보여야 한다"
```
