# /new-feature

새 기능 개발을 위한 Plan-First 템플릿.

## Phase 1 — 요구사항 파악 (코드 작성 금지)

다음 질문에 답한 후 계획 작성:

1. **어느 레이어가 영향받는가?**
   - 프론트만 / 백엔드만 / 풀스택
2. **영향받는 파일 목록은?**
   - `Glob` 또는 `Grep`으로 관련 파일 파악
3. **API 변경이 필요한가?**
   - 새 엔드포인트 / 기존 스키마 변경 / 캐시 TTL 조정
4. **디자인 토큰 변경이 필요한가?**
   - 변경 금지. 기존 토큰으로 해결 불가 시 사용자에게 명시적 승인 요청

## Phase 2 — 계획 작성

아래 형식으로 계획을 작성해 사용자에게 보여준다:

```
## 구현 계획: [기능명]

### 영향 파일
- backend/routers/xxx.py
- frontend/index.html (섹션 설명)

### 작업 순서
1. [백엔드] ...
2. [프론트] ...
3. [배포] ...

### 완료 기준
- [ ] Netlify URL: [검증 방법]
- [ ] Render health: 정상
- [ ] UI: [탭/버튼/텍스트 확인 항목]
```

**⛔ 사용자 승인 전까지 코드 작성 금지**

## Phase 3 — 구현 (승인 후)

1. Todos 생성
2. 백엔드 → 프론트 순서로 구현
3. 각 Todo 완료 즉시 체크
4. `/deploy` 커맨드 실행

## Phase 4 — 검증

Netlify 라이브 URL fetch로 완료 기준 전체 확인:
```bash
curl -s https://connecting-dots-ctd.netlify.app | python -c "..."
```
모든 기준 통과 시에만 완료 처리.
