# 커넥팅닷 (Connecting the Dots) — 모바일 웹앱 UX/UI 브리핑
**버전:** v1.0  
**작성일:** 2026년 6월 4일  
**대상:** UX/UI 디자이너  
**산출물 형식:** Figma 파일

---

## 1. 프로젝트 한 줄 정의

> 한국 개인 투자자를 위한 실시간 투자 판단 웹앱 — 모바일 브라우저(375px~) 기준 UI 설계 요청

기존 데스크톱 웹(investiq_web_v3.html)이 완성되어 있으며, 이번 작업은 **동일한 디자인 시스템 위에서 모바일 경험을 재설계**하는 것입니다. 새로운 디자인 언어를 만드는 것이 아닙니다.

---

## 2. 확정된 것 — 변경 불가

디자이너가 재해석하거나 수정하면 안 되는 항목입니다.

### 2.1 디자인 시스템 토큰 (전체 확정)

```css
/* 배경 4단계 레이어 */
--bg:  #000000;
--l1:  #0a0a0a;
--l2:  #111113;
--l3:  #1a1a1c;

/* 글래스 표면 */
--g1:  rgba(255,255,255,.04);
--g2:  rgba(255,255,255,.07);
--g3:  rgba(255,255,255,.12);
--gb:  rgba(255,255,255,.08);
--gb2: rgba(255,255,255,.15);

/* 텍스트 */
--t1:  rgba(255,255,255,1.00);
--t2:  rgba(255,255,255,0.62);
--t3:  rgba(255,255,255,0.36);
--t4:  rgba(255,255,255,0.18);

/* 강조색 — 단 하나 */
--ac:  #0a84ff;
--acd: rgba(10,132,255,.12);
--acb: rgba(10,132,255,.25);

/* 시맨틱 상태 */
--gr:  #30d158;   --grd: rgba(48,209,88,.10);
--ye:  #ffd60a;   --yed: rgba(255,214,10,.08);
--re:  #ff453a;   --red: rgba(255,69,58,.08);
--or:  #ff9f0a;
--pu:  #bf5af2;   --pud: rgba(191,90,242,.12);
```

### 2.2 폰트 역할 분담 (변경 불가)

| 폰트 | 용도 |
|---|---|
| Inter + Noto Sans KR | 모든 텍스트·문장 |
| JetBrains Mono | 모든 숫자·가격·지표값 예외 없이 |

### 2.3 배지 시스템 (변경 불가)

| 클래스 | 색상 | 사용처 |
|---|---|---|
| `.t-core` | Blue 계열 | CTD 핵심 노드 |
| `.t-bull` | Green 계열 | 강세·매수·유효 |
| `.t-warn` | Yellow 계열 | 경고·주의·과열 |
| `.t-risk` | Red 계열 | 위험·손절 |
| `.t-act`  | Orange 계열 | 행동·주목 |
| `.t-hold` | Gray 계열 | 관망·중립 |

배지 공통 스펙: `font-size 10px · font-weight 500 · padding 1px 7px · border-radius 4px · border .5px`

### 2.4 핵심 원칙 (변경 불가)

```
✅ 다크 전용 — Light mode 없음
✅ 강조색 1개 (#0a84ff) — 선택 상태·링크·스코어에만
✅ 동시 노출 최대 8개 (Miller's Law)
✅ 그라데이션·그림자·글로우 금지
✅ 레이어 색상 차이로만 깊이 표현
✅ CTD 체인 — 수평 방식 B (수직 아코디언 금지)
```

### 2.5 정보 구조 (변경 불가)

```
탭 3개: 시장 / 종목 / 나의 투자
64개 지표 → 화면에 나열 금지, 판단 출력값만 노출
레이더 8축 → 종목 평가 8항목 1:1 매핑 유지
```

## 3. 모바일 환경 확정 사항

### 3.1 타겟 기기

```
기준: 375px (iPhone SE / 13 mini — 가장 좁은 환경)
최대: 430px (iPhone Pro Max)
방식: 375px에서 설계, 430px까지 자연스럽게 확장
단위: rem 기반 유연 레이아웃 권장
```

### 3.2 레이아웃 구조

```
[Topbar 44px — sticky]
  로고 "CTD" + 현재 탭명 + 갱신 시각

[Ticker 띠 26px — sticky, Topbar 바로 아래]
  S&P500 · 코스피 · VIX · USD/KRW 좌→우 흐름
  배경: --l1, 텍스트: JetBrains Mono 11px

[Content 영역 — scroll]
  탭별 1열 레이아웃

[Bottom Nav 56px — fixed bottom]
  4탭: 시장 · 종목 · 투자 · 더보기
```

### 3.3 Bottom Nav 4탭 확정

| 탭 | 아이콘 방향 | 포함 내용 |
|---|---|---|
| 🌡️ 시장 | 온도계 | 온도 게이지·CTD·수급·국면 |
| ◎ 종목 | 원형 타겟 | 종목 칩 리스트 + Zone 4 판단 카드 통합 |
| ◈ 투자 | 다이아몬드 | 포트폴리오·세금·AI |
| ⋯ 더보기 | 점 3개 | 설정·데이터 신선도·알림 |

**아이콘 스펙:** 24×24px, stroke 기반, 활성 = --ac 색상, 비활성 = --t3

## 4. 탭별 모바일 재설계 명세

### 4.1 시장 탭

**섹션 순서 (위→아래 1열 스택):**

```
① Hero — 온도 게이지
   반원형 아크, 모바일 크기로 축소
   온도 숫자: 72px JetBrains Mono (웹 120px → 축소)
   판단 문장: 16px Inter 700
   퀵 메트릭 칩 6개: 2열 3행 그리드

② CTD 체인 (Q1 확정: 가로 스크롤)
   노드 5개 가로 나열
   컨테이너 overflow-x scroll, 스냅 포인트
   우측 끝 페이드아웃으로 "더 있음" 힌트
   활성 노드 아래 상세 패널 펼침

③ 수급 바
   외국인·기관·개인 방향 바
   판정 문장 1줄

④ 종목 신호 요약
   국내/해외 Top5 평균 스코어
   #1 종목 하이라이트
```

### 4.2 종목 탭

**섹션 구조:**

```
① 시장 컨텍스트 브릿지
   현재 온도 + 국면 배지 + CTD 딥링크 (1줄)

② 종목 칩 가로 스크롤 (웹 사이드바 대응)
   국내/해외 토글
   종목 칩: 아이콘 + 종목명 + IQ스코어
   선택된 칩 = --ac 강조
   overflow-x scroll

③ Zone 4 판단 카드 (칩 선택 즉시 교체)
   a. 종목명·가격·강약점 배지 헤더
   b. CTD 인과관계 (가로 스크롤 체인)
   c. 레이더 차트 260px (Q3 확정)
      레전드 2열 그리드 아래 배치
   d. 투자 논거 유효성 목록
   e. 매수 플랜
      레인지 바 → 테이블 → 목표가/손절 → 세후 비교 카드
```

**웹 패턴 → 모바일 번역:**
```
웹:     사이드바 종목 클릭 → 오른쪽 판단 카드 즉시 교체
모바일: 상단 종목 칩 탭   → 하단 판단 카드 즉시 교체
        (같은 탭 안, 탭 전환 없음)
```

### 4.3 투자 탭

```
웹 2열 → 모바일 1열 순서:

① 포트폴리오 통계 칩 (2×2 그리드)
② 포트폴리오 배분 리스트
   국내 = [세후 = 세전] 녹색 배지
   해외 = [세금 22%] 빨간 배지
③ 세금 계산기
   입력 → 결과 (세전→세후 흐름)
④ AI 투자 분석
   컨텍스트 칩 3개 + 입력창 + 예시 질문
```

### 4.4 더보기 탭

```
① 데이터 신선도 상태
② 알림 설정
③ 앱 정보
(상세 설계는 디자이너 자율 — 단, 디자인 토큰 준수)
```

## 5. 컴포넌트 모바일 적용 규칙

### 5.1 터치 타겟

```
최소 터치 영역: 44×44px (Apple HIG 기준)
종목 칩 높이: 44px 이상
Bottom Nav 탭 영역: 전체 높이 56px
```

### 5.2 레이더 차트 (Q3 확정)

```
크기: 260×240px 유지 (웹과 동일)
이유: SVG 코드 재사용 + 343px 가용 너비에서 여백 충분
가용 너비 계산: 375px - 패딩 16px×2 = 343px
레이더 260px → 양쪽 여백 각 41.5px
```

### 5.3 CTD 가로 스크롤 체인 (Q1 확정)

```
노드 너비: 120px 고정
노드 간격: 화살표 포함 160px
전체 체인: 5노드 × 160px = 800px (컨테이너 초과 → 스크롤)
스냅: scroll-snap-type x mandatory
힌트: 우측 끝 40px에 linear-gradient 페이드아웃
활성 노드 = --acd 배경 + --ac 텍스트
```

### 5.4 타이포그래피 모바일 스케일

| 용도 | 웹 | 모바일 |
|---|---|---|
| 시장 온도 숫자 | 120px | 72px |
| 종목명 (Zone 4 헤더) | 40px | 28px |
| 섹션 타이틀 | 20px | 16px |
| 카드 수치 | 16px | 14px |
| 본문 | 14px | 13px |
| 보조·배지 | 11~12px | 11px (유지) |

## 6. 미결 사항 — 디자이너 판단 영역

아래 항목은 IA/로직 결정이 완료되었으며, **시각적 표현 방식은 디자이너가 결정**합니다.

```
① Bottom Nav 아이콘 최종 형태
   방향만 정의됨 (온도계·타겟·다이아몬드·점3개)
   → 실제 stroke 아이콘 디자인은 디자이너 결정

② 종목 칩 선택 활성 상태 애니메이션
   색상 기준만 정의됨 (--ac)
   → 전환 방식(fade/slide/scale) 디자이너 결정

③ Zone 4 섹션 간 구분 방식
   구분선 vs 여백 vs 배경색 차이
   → 디자이너 결정 (단, 그림자 금지)

④ 더보기 탭 레이아웃 전체
   콘텐츠 목록만 정의됨
   → 디자이너 자율 설계
```
